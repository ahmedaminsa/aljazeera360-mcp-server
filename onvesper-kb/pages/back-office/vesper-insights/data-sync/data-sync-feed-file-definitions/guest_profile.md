# Guest\_Profile

The guest profile is a consolidation of details for every guest user on the ES platform. A guest user is someone who accesses the service, provides consent or user preference information, and may or may not have watched content that did not require any purchase or registration.

Only one record exists for each guest user in this table.

Each guest profile record holds the latest state of a user.

The Guest Profile contains many fields that aggregates their behavior on the platform, including but not limited to the following:

* Aggregated viewing statistics
* Consent and preference settings

### FAQ <a href="#faq.5" id="faq.5"></a>

This section attempts to answer frequent questions about data you find in this guest\_profile data sync feed

#### Does the guest profile show users who converted (i.e., registered on the platform after being a guest user)?

Yes, the guest profile includes data for users who initially accessed the platform as guest users and later registered. When a user converts, the field `is_converted` is set to true, and the profile is updated to reflect their conversion, while still capturing their earlier guest interactions.

#### Does the guest profile include data for users who haven't watched any content?

Yes, the guest profile includes users who may have provided consent or preference information, even if they haven't watched any content. The profile captures data regardless of whether the user has interacted with content that requires no purchase or registration.

#### Can a guest user have multiple records in the guest profile table?

No, each guest user has only one record in the guest profile table. This record reflects the latest state of the user's interactions, preferences, and consent information on the platform.
