# Customer\_Profile

The customer profile is a customer level detail of every user on the ES platform. A user can either be:

* a customer (someone who has made purchases, whether a single transaction like a PPV or a subscription, regardless of its current status)
* a registrant (someone who registered without making a purchase)
* a guest-user (someone who access the service and provided consent or user preference information and that may or may not watched content that did not require any purchase or registration)

Only one record exists for each user in this table.

Each customer profile record holds the latest state of a user.

The Customer Profile contains many fields that aggregates their behavior on the platform, including but not limited to the following:

* Aggregated viewing statistics
* Aggregated purchase statistics
* Aggregated Licence statistics
* Consent and preference settings

### FAQ

This section attempts to answer frequent questions about data you find in this customer\_profile data sync feed

1. How is the last viewing session selected when enriching the customer\_profile with the last\_watch\_xxx columns (ie: last\_watch\_content\_id, last\_watch\_duration, last\_watch\_device, etc)
   1. Only the view session’s started\_at field is used for last\_watch\_xxx columns. These columns are quite volatile in that they change the next time the user watches something. Any view session with the same started\_at can be the last watching candidate chosen to aggregate to customer profile.
