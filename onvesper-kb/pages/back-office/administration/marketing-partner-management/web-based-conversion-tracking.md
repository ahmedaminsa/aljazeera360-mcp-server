# Web-based conversion tracking

Campaigns for web-based conversion are created via Vesper Back Office.

To enable them, you have to correctly implement the redirect URL, which must direct to your web application and contain the `esrefid` parameter, which will be generated on the Vesper platform every time you create a new campaign.

## Tracking the conversion

Marketing Partner Management will track conversion against an end-user when they click on a specific redirect URL, which initialises the marketing session, and complete the conversion within the attribution window (e.g. seven days from campaign creation) set up against the campaign.&#x20;

## Reporting metrics

The **Referral Partner Name** and **Campaign Name** will appear in Vesper reporting based on your campaign inputs on the Marketing Partner Management dashboard.&#x20;

The **Referral ID** will be a unique string generated by Vesper that maps to a given Campaign.

**Click ID** is an optional feature and will appear when implemented.&#x20;

You can find this information on all reports that showcase order management (for purchases) and customer profile data (for signups).&#x20;

## Referral ID

The `esrefid` string parameter stands for '**E**ndeavor **S**treaming **Ref**erral **ID**', which is needed to start the tracking from the moment a user clicks on a specific redirect URL that takes them to the resident’s web app hosted by ES.&#x20;

## Campaign creation in Vesper Back Office

<figure><img src="/files/NjkxgmCazXT6lDrg14TB" alt=""><figcaption></figcaption></figure>

1. Go to Vesper Back Office -> Administration -> Marketing Partner Management
2. Go to 'Create New Campaign' on the top right of the dashboard
3. Give the campaign a name
4. Name the Referral Partner
5. Select the conversion event type (signup and/or purchase)
6. Select the attribution window - the timeframe when to count conversion, which can be days or hours
7. As an **optional** feature, you can enable a Post-Back URL, which will need to be provided by your Campaign partner according to their documentation.&#x20;
8. Hit Save
9. A new unique `esrefid` will be generated, which you'll have to append to your redirect URL.

## Anatomy of a redirect URL

<figure><img src="/files/l19XFKeIKbcJwrI3q76x" alt=""><figcaption><p>Different sections of a Redirect URL used by your referral partners</p></figcaption></figure>

The redirect URL consists of two mandatory sections followed by an optional one.

* Domain section (mandatory) - this is your service domain
* `esrefid` (mandatory) - this is the unique parameter generated by Endeavor when the campaign is created in Vesper Back Office and then appended to a domain. It does not need to be the first parameter in the URL -you can combine it with any other parameters- but it needs to be present for the conversion to be tracked.&#x20;
* `esclickid` (optional) - this is a fixed Endeavor parameter that captures an additional data point. Its value will be dynamic and will represent a specific user or click data.&#x20;

## Combination of UTM and `esrefid` parameters

UTM (Urchin Tracking Module) parameters are popular text tags attached to a URL to track source, medium and campaign information. They are specifically used with Google Analytics for the performance of online marketing campaigns against various traffic sources.

You don't have to sacrifice one parameter *in lieu of* using another one; they can work together!

Below are a couple of examples of a URL containing both UTM and `esrefid` parameters, in whichever order you prefer (just note that `?` is added right before the first parameter and multiple parameters are then separated by an ampersand (`&`) :

`mydomain.com/signup?esrefid=1234&utm=abcd`

`mydomain.com/signup?utm=abcd&esrefid=1234`
